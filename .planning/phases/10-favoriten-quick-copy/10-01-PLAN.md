CREATE TABLE IF NOT EXISTS favoriten_fahrten (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  von_ort_id INT NOT NULL,
  nach_ort_id INT NOT NULL,
  anlass VARCHAR(500) NOT NULL,
  abrechnungstraeger_id INT NOT NULL,
  sort_order INT DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (von_ort_id) REFERENCES orte(id) ON DELETE CASCADE,
  FOREIGN KEY (nach_ort_id) REFERENCES orte(id) ON DELETE CASCADE,
  FOREIGN KEY (abrechnungstraeger_id) REFERENCES abrechnungstraeger(id) ON DELETE CASCADE
);

1. Controller favoritController.js (per D-04, D-05):
- `getAllFavoriten(req, res)` — ruft FavoritFahrt.findAll(req.user.id) auf, gibt JSON-Array zurueck
- `createFavorit(req, res)` — validiert Body mit createFavoritSchema.parse(), ruft FavoritFahrt.create() auf, gibt 201 mit neuem Favorit zurueck
- `deleteFavorit(req, res)` — ruft FavoritFahrt.delete(req.params.id, req.user.id), gibt 200 zurueck. Wenn Favorit nicht gefunden: 404
- `executeFavorit(req, res)` — (per D-05, D-09):
  a) Favorit laden via FavoritFahrt.findById(req.params.id, req.user.id)
  b) Falls nicht gefunden: 404
  c) Kilometer berechnen via getDistance(favorit.von_ort_id, favorit.nach_ort_id, req.user.id)
  d) Neue Fahrt erstellen via Fahrt.create() mit: datum = heutiges Datum (YYYY-MM-DD), anlass = favorit.anlass, vonOrtId = favorit.von_ort_id, nachOrtId = favorit.nach_ort_id, abrechnung = favorit.abrechnungstraeger_id, kilometer = berechnete km (oder 0 falls keine Distanz), userId = req.user.id
  e) 201 mit erstellter Fahrt-ID zurueckgeben
- Fehlerbehandlung: try/catch mit console.error und 500er-Response, deutsche Fehlermeldungen
