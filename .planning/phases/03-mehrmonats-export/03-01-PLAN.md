```sql
WHERE f.datum >= ? AND f.datum < ? AND f.user_id = ?
```

- `startDate = '${startYear}-${String(startMonth).padStart(2,'0')}-01'`

Die Methode gibt die gleiche Spaltenstruktur zurueck wie `getMonthlyReport` (inklusive Mitfahrer-JOINs).

In `backend/utils/excelExport.js` eine neue Funktion `exports.exportToExcelRange` erstellen. Diese Funktion ist eine Kopie von `exports.exportToExcel` mit folgenden Aenderungen:

1. **Parameter extrahieren:** `const { startYear, startMonth, endYear, endMonth, type } = req.params;`

2. **Daten laden:** `Fahrt.getDateRangeReport(startYear, startMonth, endYear, endMonth, userId)` statt `getMonthlyReport`.

3. **Header-Zeitraum (per D-07, D-08, D-09):**
   - Pruefen ob Einzelmonat: `startYear === endYear && startMonth === endMonth`
   - Einzelmonat: `getMonthName(parseInt(startMonth)) + ' ' + startYear` (wie bisher, per D-08)
   - Mehrmonat: `String(startMonth).padStart(2,'0') + '/' + startYear + ' - ' + String(endMonth).padStart(2,'0') + '/' + endYear` (per D-07, D-09)
   - Diesen Wert in `abrechnungWorksheet.getCell('B2')` und ggf. `mitnahmeWorksheet.getCell('B2')` setzen.

4. **Eingereicht-Status (per D-04, D-05):**
   Am Ende der Funktion (nach erfolgreichem Response-Send), fuer jeden Monat im Zeitraum `Abrechnung.updateStatus()` aufrufen:
   ```javascript
   // Status-Update fuer jeden Monat im Zeitraum
   const Abrechnung = require('../models/Abrechnung');
   let y = parseInt(startYear), m = parseInt(startMonth);
   const ey = parseInt(endYear), em = parseInt(endMonth);
   while (y < ey || (y === ey && m <= em)) {
     await Abrechnung.updateStatus(userId, y, m, type, 'eingereicht', new Date().toISOString().slice(0, 10));
     m++;
     if (m > 12) { m = 1; y++; }
   }
   ```
   WICHTIG: Das Status-Update VOR dem Response senden, damit bei Fehler ein 500 zurueckkommt.

5. **Dateiname:** `fahrtenabrechnung_${type}_${startYear}_${startMonth}_bis_${endYear}_${endMonth}.xlsx` (bzw. `.zip` bei Pagination).

6. **Bestehende `exports.exportToExcel` Funktion NICHT aendern** (per D-10, D-12). Die neue Funktion ist eine separate export-Funktion.

7. Den Abrechnung-require an den Anfang der Datei setzen: `const Abrechnung = require('../models/Abrechnung');`
