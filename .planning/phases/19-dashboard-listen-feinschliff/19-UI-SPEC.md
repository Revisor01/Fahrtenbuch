## Component Inventory

### New Components / Patterns

#### 1. Inline-Edit Mode (Dashboard + Fahrtenliste)

**Trigger:** Click Pencil icon on a Fahrt-Card.
**Behavior (D-01, D-02):** The card is replaced in-place by `FahrtForm` pre-filled with trip data. No overlay, no modal — the form occupies the same card slot.
**States:**
- Idle: Normal card display
- Editing: Card replaced by FahrtForm with pre-filled values
- Saving: Form disabled, btn-primary shows "Fahrt speichern..." (disabled state)
- Success: Form collapses, card returns with updated data

**FahrtForm Edit-Mode Props:**
- `editData`: Object with pre-filled trip values
- `onUpdate`: Callback receiving updated trip data
- `onCancel`: Callback to close edit mode, restore card

**Layout:** FahrtForm renders inside the same `space-y-3` container that held the card. Cancel button (`btn-secondary`) + Save button (`btn-primary`) at bottom.

#### 2. Fahrt-Card (shared between Dashboard + Fahrtenliste)

**Design (D-04, D-05):** Identical card layout in both views. Reuse existing Dashboard "Letzte Fahrten" card pattern:

```
+-----------------------------------------------------------+
| [Datum]  Von-Ort -> Nach-Ort              [km] km         |
| [Anlass] [Traeger]  [Mitfahrer-Badge]                     |
|                                                            |
| [Nochmal] [Rueckfahrt] [Pencil] [Trash]   (right-aligned) |
+-----------------------------------------------------------+
```

- Container: `bg-primary-25 dark:bg-primary-900 rounded-card p-4`
- Route text: `text-sm text-value font-semibold`
- Meta line: `text-xs text-label`
- Buttons area: `flex items-center gap-1` right-aligned
- Action icons: 13px Lucide icons, ghost button styling with `p-1.5 rounded-card`

**Mobile icon-only action buttons require `aria-label`:**

| Button | aria-label |
|--------|------------|
| Pencil (Edit) | `aria-label="Fahrt bearbeiten"` |
| Trash2 (Delete) | `aria-label="Fahrt loeschen"` |
| Copy (Nochmal) | `aria-label="Fahrt kopieren"` |
| ArrowLeftRight (Rueckfahrt) | `aria-label="Rueckfahrt erstellen"` |

**Fahrtenliste-spezifisch:**
- Zeigt alle Fahrten (nicht nur letzte 3)
- Sortierung beibehalten (Datum absteigend default)
- Filter-Bereich bleibt als eigene `card-container` oberhalb

#### 3. User-Card (Verwaltung-Tab)

**Design (D-07, D-08, D-10):** Card-Layout konsistent mit Settings Sub-Tabs.

```
+-----------------------------------------------------------+
| [Avatar-Initialen]  Username                   [Rolle-Badge] |
|                     email@example.de                        |
|                     Kirchengemeinde XY                      |
|                                                              |
| [Status-Badge: Verifiziert/Nicht verifiziert]               |
|                                                              |
| [Bearbeiten]                              [Loeschen]        |
+-----------------------------------------------------------+
```

- Container: `card-container` (standard card with hover)
- Avatar: 40x40px rounded-full div, `bg-primary-100 text-primary-700 font-semibold`, zeigt Initialen (erste 2 Buchstaben des Usernamens)
- Name: `text-sm font-semibold text-value`
- Email: `text-xs text-label`
- Kirchengemeinde: `text-xs text-label`
- Rolle-Badge: `status-badge-primary` fuer Admin, `status-badge` mit `bg-primary-50 text-primary-600` fuer User
- Status-Badge: `status-badge-primary` fuer verifiziert (mit CheckCircle2 icon 12px), orange/secondary fuer nicht verifiziert
- Bearbeiten: `btn-secondary` mit Pencil icon, `aria-label="Benutzer bearbeiten"`
- Loeschen: `btn-destructive` mit Trash2 icon (nur wenn nicht eigener User), `aria-label="Benutzer loeschen"`
- Grid: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4`

#### 4. Chart-Animation (Statistik)

**Design (D-11):** CSS-basierte Eingangsanimation fuer Balkendiagramm-Bars.

```css
@keyframes barGrow {
  from { transform: scaleY(0); }
  to { transform: scaleY(1); }
}
```

- Origin: `transform-origin: bottom`
- Duration: 400ms
- Easing: `ease-out`
- Stagger: Each bar delayed by `i * 50ms` (via inline style `animation-delay`)
- Trigger: On mount / when `statistikJahr` changes

## Interaction Contracts

### Inline-Edit Flow (Dashboard)

| Step | User Action | System Response |
|------|------------|-----------------|
| 1 | Clicks Pencil on trip card | Card fades out (150ms), FahrtForm fades in (150ms) pre-filled |
| 2 | Edits fields | Standard form interaction |
| 3a | Clicks "Fahrt speichern" | Form disabled, API call, success notification, form collapses, updated card appears |
| 3b | Clicks "Abbrechen" | Form collapses, original card restored |

**Transition:** Use existing `tab-content-fade` animation (150ms ease-out) for card-to-form and form-to-card transitions.

### Inline-Edit Flow (Fahrtenliste)

Identical to Dashboard flow (D-06). Same FahrtForm component, same props, same animation.

### Delete Flow (existing, no change)

| Step | User Action | System Response |
|------|------------|-----------------|
| 1 | Clicks Trash icon | showNotification confirm dialog |
| 2a | Confirms | Trip deleted, list refreshes |
| 2b | Cancels | Dialog closes, no action |

### User Management Actions

| Step | User Action | System Response |
|------|------------|-----------------|
| 1a | Clicks "Bearbeiten" on user card | Modal opens with UserForm pre-filled (keep existing modal pattern) |
| 1b | Clicks "Loeschen" on user card | showNotification confirm: "Benutzer [username] wirklich loeschen?" |
| 2 | Clicks "+ Neuer Benutzer" | Modal opens with empty UserForm |

## Responsive Behavior

### Fahrt-Cards (Mobile < 640px)

- Full width, stacked vertically with gap-3
- Action buttons: icon-only (text labels hidden via `hidden sm:inline`, existing pattern)
- All icon-only buttons MUST have `aria-label` (see Component Inventory section 2)
- Touch targets: min 44x44px for all buttons
- Route text wraps naturally

### Fahrt-Cards (Desktop >= 640px)

- Full width within card-container
- Action buttons show text labels ("Nochmal", "Rueckfahrt")
- Horizontal layout for meta info

### User-Cards (Mobile < 640px)

- Single column: `grid-cols-1`
- Full-width cards
- Buttons stack: `flex-col`
- All icon-only buttons MUST have `aria-label` (see Component Inventory section 3)

### User-Cards (Tablet >= 640px)

- Two columns: `sm:grid-cols-2`

### User-Cards (Desktop >= 1024px)

- Three columns: `lg:grid-cols-3`

### Inline-Edit Form (Mobile)

- FahrtForm already responsive (form-row, form-group classes)
- Cancel + Save buttons: `flex-col sm:flex-row gap-2`, full width on mobile
