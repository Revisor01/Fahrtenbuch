In backend/controllers/fahrtController.js, Funktion getMonthlyOverview (ca. Zeile 450-475):

1. Die Zeile `const mitfahrerErstattung = row.mitfahrer_count * 0.05 * row.kilometer;` (Zeile 466) ersetzen.

2. Die SQL-Query ab Zeile 395 liefert bereits `eb.betrag` via JOIN mit mitfahrer_erstattung. Das Problem: fuer Mitfahrer-Zeilen wird `row.betrag` nicht genutzt, sondern stattdessen 0.05 hardcoded.

3. Loesung: Die `row.betrag` wird bereits korrekt aus dem JOIN geliefert (die UNION-Query schliesst mitfahrer_erstattung ein). Ersetze die hardcoded Berechnung durch:
   ```javascript
   const mitfahrerErstattung = row.mitfahrer_count * row.betrag * row.kilometer;
   ```
   ABER: `row.betrag` bezieht sich auf den Traeger-Erstattungssatz, nicht auf den Mitfahrer-Satz.

4. Bessere Loesung: Eine separate Mitfahrer-Erstattungsberechnung einbauen, analog zu exportToExcel.
   - Vor der reduce-Schleife: Mitfahrer-Erstattungssaetze aus DB laden:
   ```javascript
   const [mitfahrerSaetze] = await db.execute(
     'SELECT betrag, gueltig_ab FROM mitfahrer_erstattung WHERE user_id = ? ORDER BY gueltig_ab DESC',
     [userId]
   );
   ```
   - Helper-Funktion fuer Mitfahrer-Satz analog zu getErstattungssatz:
   ```javascript
   const getMitfahrerSatz = (datum) => {
     const passend = mitfahrerSaetze.find(s => new Date(s.gueltig_ab) <= new Date(datum));
     if (!passend && mitfahrerSaetze.length > 0) return mitfahrerSaetze[mitfahrerSaetze.length - 1].betrag;
     return passend ? passend.betrag : 0;
   };
   ```
   - In der reduce-Schleife (Zeile 466) ersetzen:
   ```javascript
   const mitfahrerErstattung = row.mitfahrer_count * getMitfahrerSatz(row.datum) * row.kilometer;
   ```

5. Den TODO-Kommentar auf Zeile 465 entfernen.

Per BUG-01: Hardcoded 0,05 EUR/km ersetzen durch DB-Lookup aus mitfahrer_erstattung Tabelle.

**BUG-02 — Rueckfahrt-Matching (per BUG-02):**

In `findErgaenzendeFahrt` (ca. Zeile 747):

1. Das bestehende Matching nutzt bereits `toLowerCase()` und `parseInt()` fuer IDs — das ist gut.

2. Fallback hinzufuegen wenn ID-Matching fehlschlaegt: Orts-Namen-Matching.
   Nach dem ID-basierten `find()` (Zeilen 795-799 fuer Hinfahrt, 807-811 fuer Rueckfahrt),
   falls kein Ergebnis, einen zweiten find() mit Orts-Namen-Vergleich:

   Fuer Rueckfahrt-Fall (istRueckfahrt === true), nach Zeile 803:
   ```javascript
   if (!hinfahrt) {
     // Fallback: Matching ueber Orts-Namen
     const hinfahrtByName = fahrtenAmSelbenTag.find(f =>
       !f.anlass?.toLowerCase().includes('rückfahrt') &&
       f.von_ort_name?.toLowerCase() === aktuellefahrt.nach_ort_name?.toLowerCase() &&
       f.nach_ort_name?.toLowerCase() === aktuellefahrt.von_ort_name?.toLowerCase()
     );
     if (hinfahrtByName) return hinfahrtByName;
   }
   ```

   Fuer Hinfahrt-Fall, analog nach Zeile 816:
   ```javascript
   if (!rückfahrt) {
     const rückfahrtByName = fahrtenAmSelbenTag.find(f =>
       f.anlass?.toLowerCase().includes('rückfahrt') &&
       f.von_ort_name?.toLowerCase() === aktuellefahrt.nach_ort_name?.toLowerCase() &&
       f.nach_ort_name?.toLowerCase() === aktuellefahrt.von_ort_name?.toLowerCase()
     );
     if (rückfahrtByName) return rückfahrtByName;
   }
   ```

**BUG-03 — Axios Interceptor Race Condition (per BUG-03):**

In `setupAxiosInterceptors` (ca. Zeile 122):

1. Ein Flag `isLoggingOut` als useRef hinzufuegen (oben bei den anderen useRef/useState, ca. Zeile 30-50):
   ```javascript
   const isLoggingOut = useRef(false);
   ```

2. Den Interceptor aendern:
   ```javascript
   const setupAxiosInterceptors = () => {
     axios.interceptors.response.use(
       (response) => response,
       (error) => {
         if (error.response && error.response.status === 401) {
           if (!isLoggingOut.current) {
             isLoggingOut.current = true;
             logout();
           }
         }
         return Promise.reject(error);
       }
     );
   };
   ```

3. In der `logout` Funktion am Ende `isLoggingOut.current = false;` setzen,
   damit nach einem Logout + erneutem Login das Flag zurueckgesetzt wird.

4. Den `console.log('Unauthorized, logging out...');` entfernen.
