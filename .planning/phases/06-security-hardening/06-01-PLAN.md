**backend/app.js** — Drei Aenderungen:

1. Oben bei den requires hinzufuegen:
```javascript
const helmet = require('helmet');
```

2. NACH `app.use(cors(...))` und VOR `app.use(express.json())` einfuegen (SEC-01):
```javascript
app.use(helmet());
```
Keine spezielle CSP-Konfiguration noetig — Frontend wird ueber separaten Nginx-Container ausgeliefert, nicht ueber Express. helmet() Defaults sind ausreichend.

3. `express.json()` Zeile aendern zu (SEC-04):
```javascript
app.use(express.json({ limit: '10mb' }));
```

**backend/routes/auth.js** — Rate-Limiter auf Login (SEC-02):

1. Oben hinzufuegen:
```javascript
const rateLimit = require('express-rate-limit');

const loginLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 Minuten
  max: 10, // max 10 Versuche pro IP
  message: { message: 'Zu viele Login-Versuche. Bitte in 15 Minuten erneut versuchen.' },
  standardHeaders: true,
  legacyHeaders: false,
});
```

2. Login-Route aendern:
```javascript
router.post('/login', loginLimiter, authController.login);
```

Register-Route NICHT rate-limiten — wird selten genutzt und hat eigene Validierung.
